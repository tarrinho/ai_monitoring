Settings

Operator tuning applied live (no restart), persisted across restarts, and overriding the .env defaults. Only non-secret operational settings are here — secrets, ports, backend URLs and security switches stay in .env. Reset returns a setting to its environment/default value. Changes are admin-only and audited.

Teams

Assign each LiteLLM key to a team and set its monthly budget ($/mo) for the Spend & Quota by-team rollup. LiteLLM reports a key's team, but team budgets are a LiteLLM Enterprise feature — so both are managed here. The values below override what LiteLLM reports (team) and its max_budget (budget); Reset clears both for that key. Detected teams are cached (LiteLLM's lookup is flaky) — the board no longer re-fetches on every load; use Refresh to re-detect.

Model costs

Each model's cost is either real (external paid API — a market price LiteLLM tracks) or estimated (self-hosted / reference rate). This drives the real-vs-estimated split on Spend & Quota. The kind is auto-detected from the model name — override it here when the guess is wrong (e.g. an open-weight model you route through a paid API). Reset returns a model to auto-detect.
